# Anvesha — How It Works (Technical Overview)

A plain-language + technical explanation of the whole project, so you can
confidently answer "what is this and how did you build it?"

---

## 1. What it is (the pitch)

**Anvesha** is an interactive 3D web experience that turns PICT's magazine
*PICTOREAL Volume 28* into an explorable world. Instead of scrolling a PDF, you
walk a lamp-lit underground stepwell — shaped like the magazine's mandala-eye
logo — as the **Sutradhar** (a turbaned guide, "the holder of the thread").
Every one of the **154 magazine pages** is hidden as a glowing tablet in the
cave; walk up to one and it opens, and the Sutradhar narrates it aloud in its
own language (English / Hindi / Marathi). Theme: *Anvesha* = "the seeking."

It runs in any browser, on desktop and phone, at a public link. No install.

---

## 2. Tech stack (what was used)

| Layer | Tech | Why |
|---|---|---|
| Language | Vanilla **JavaScript** (ES modules) | No framework needed; keeps it light and fast |
| 3D engine | **Three.js** (r0.185) | WebGL rendering — the cave, avatar, lighting, camera |
| Build tool | **Vite** 5 | Instant dev server + optimized production bundle |
| Tests | **Vitest** 2 | 20 unit tests on the pure logic (world gen, state) |
| UI animation | **GSAP** | Smooth transitions for the reader/journal/HUD |
| Audio | **Web Audio API** + **HTMLAudioElement** | Synth fallback + music/voice playback |
| Speech | **ElevenLabs** (pre-rendered) + **Web Speech API** (fallback) | The Sutradhar's voice |
| Hosting | **GitHub Pages** + **GitHub Actions** | Free static hosting, auto-deploy on push |
| Media tooling | **ffmpeg** | Image/video optimization (offline, build-time) |

**Key architectural choice:** the shipped game is a **fully static site** —
just HTML/JS/CSS + media files. There is no server, no database, no login.
Everything runs in the visitor's browser. That's why it's free to host and
essentially unbreakable.

## 3. How the world is built

### Procedural cave (`src/world/cave.js`)
The cave layout isn't hand-placed — it's **generated by code**, deterministically
(same every load, no randomness), in the shape of the Pictoreal mandala-eye:

- a central **iris hall** (the emblem), a ring of **mandala petals**, an
  almond **eye-outline** corridor, **eyelash** spokes, a **lotus** crown chamber,
  and an outer **ring** — all as a graph of *nodes* (chambers) + *edges* (tunnels).
- It produces ~250 **niches** (page slots) along the walls.
- This module is **pure** (no Three.js, no browser) so it can be **unit-tested** —
  that's what most of the 20 tests check (the hall exists, the graph is
  connected, the entrance is clear, niche count, etc.).

### Collision (also in `cave.js`)
No physics engine. Movement uses **analytic collision**: a point is "walkable"
if it's inside any chamber circle, near any tunnel segment, or in the entrance
corridor (`isWalkable`). Movement slides along walls (`slideMove`). Cheap and
predictable — no clipping through rock.

### The 3D scene (`src/game3d.js`, ~1400 lines — the heart of the game)
Builds everything from the cave data: floor/walls/arches (merged geometries for
performance), the carved gate with the Saraswati mural, the descending stairs,
the floating Pictoreal emblem, instanced page-plaques, a glowworm "sky" and a
mandala-eye constellation overhead, dust motes, and all the lighting (warm
lamp-lit temple, not a cold crypt). The Sutradhar's **lamp** is a real
point-light that reveals plaques from darkness as you approach.

## 4. The Sutradhar (avatar)

- The character was **generated from concept art** using **TRELLIS** (a free
  image-to-3D AI model, run on a Hugging Face Space) — a single reference image
  became a textured `.glb` 3D model (`public/models/sutradhar.glb`).
- The game auto-loads it, scales it, stands it on the floor, and keeps the lamp.
- Since the mesh has no skeleton, the walk is **faked in code** the stylized way:
  a step-hop, forward lean, foot-to-foot roll, and squash-and-stretch — reads as
  walking at game distance without needing an animation rig.

## 5. The pages & the reader

- Source: 154 real magazine scans (1.1 GB of PNG/JPG). **ffmpeg** optimized them
  to ~1400px JPEGs (76 MB total) for the reader, plus tiny **192px thumbnails**
  (2.5 MB) shown on the cave plaques. Originals never go in the repo.
- A generator script (`scripts/build-real-pages.mjs`) reads a manifest and
  produces `src/content/real-pages.js` — the page list, grouped into 10 real
  sections (Opening, Blessings, Faculty, Batch, English, Hindi, Arts, Marathi,
  Events, Clubs), each tagged with a language.
- Walking to a plaque (or clicking it, or the Index) opens the **reader**
  (`src/ui/reader.js`) — the full-res page with a "Read aloud" button and a
  spoken summary.
- **Index** (`contents.js`) = table of contents that unlocks progressively as
  you explore. **Collection** (`journal.js`) = a gallery of all 154 pages,
  found ones in colour, the rest locked — tap one to jump back to it.

## 6. The voice (narration)

Two-tier system (`src/systems/voice.js`):

1. **Primary — ElevenLabs.** Every line (welcome, section intros, and a spoken
   summary of *each* page) was pre-rendered to MP3 with the voice
   *"Sanjay – Mature Hindi Storyteller"* (multilingual — handles all three
   languages). A build script (`scripts/generate-voice.mjs`) calls the ElevenLabs
   API and writes `public/voice/*.mp3` + a manifest. **The API key stays on the
   dev machine — it never enters the repo or the browser.** Only the finished
   MP3s ship. 174 clips, ~36 MB.
2. **Fallback — Web Speech API.** If a clip is missing, the browser's built-in
   text-to-speech reads it, picking a Hindi/Marathi/Indian-English voice by the
   page's language tag.

**Where the page summaries came from:** the pages are image scans (no text
data), so the actual articles were *read* and condensed into warm 25–45-word
spoken introductions in each page's language — that's what the Sutradhar says.

## 7. Music & audio

- Ambient music: **"Desert City" by Kevin MacLeod** (Creative Commons BY 4.0,
  credited in README) loops quietly during exploration
  (`public/audio/ambient-music.mp3`).
- Fallback: a synthesized tanpura-style drone (Web Audio API) if the file fails.
- Plus footstep, chime and gate SFX. Everything respects the mute button.
- *Note:* music starts on the first tap/Space, not on the splash — browsers
  forbid audio autoplay before a user gesture.

## 8. Controls & platforms

- **Desktop:** WASD/arrows to move, mouse-drag (or pointer-lock) to look,
  E/Space/click to open, I = index, J = collection, V = camera angle, L = theme.
- **Mobile:** an on-screen **virtual joystick** + interact/camera buttons, and
  finger-drag to look. Detected automatically; the desktop hint hides on touch.
- Mobile performance is tuned down (no point-light shadows, capped pixel ratio,
  softer bloom) and the field-of-view widens in portrait.
- Movement is **frame-rate-independent** (delta-time based) so 120 Hz phones
  don't run faster than 60 Hz desktops.

## 9. The opening

- A cinematic **splash video** (the Logo_1 brand sting: dust → mandala-eye
  reveal → Pictoreal logo) plays muted and looping behind the ANVESHA title.
- Press Space / tap → the camera swoops down into the well, the Sutradhar speaks
  the welcome, and you take control.

## 10. Deployment (how it goes live)

- Code lives on **GitHub**: `github.com/Mahesh-Loya/anvesha-pictoreal`.
- A **GitHub Actions** workflow (`.github/workflows/deploy.yml`) runs on every
  push to `master`: it installs deps, **runs the tests**, builds with Vite, and
  publishes `dist/` to **GitHub Pages**.
- Live at: **https://mahesh-loya.github.io/anvesha-pictoreal/**
- Asset URLs are relative so the site works from the `/anvesha-pictoreal/`
  subpath. There's an Open Graph card + favicon so shared links preview nicely.

## 11. Notable problems solved (good "war stories" to mention)

- **Deterministic mandala layout** so the world is identical every load and
  unit-testable.
- **AI-generated 3D character** from 2D art (TRELLIS), animated procedurally
  since it has no skeleton.
- **Read-aloud in 3 languages** with the right voice per language, and a
  double-voice race on mobile fixed with a manifest-gate + generation guard.
- **1.1 GB of scans → 76 MB** web-ready without losing legibility.
- **Frame-rate-independent movement** and a "phantom ramp" bug (avatar flying)
  fixed by making ground-height depend on position, not just depth.
